jQuery Events Methods (main used)

click()

$('h1').click(function(){ alert("H1 Clicked!"); });

$('#submit').click(function(){ console.log('something was clicked'); });

$("button").click(function({ alert("YOU CLICKED IT!"); });

$("button").click(function(){ $(this).css("background","pink");});

$("button").click(function(){ console.log("you clicked " + $(this).text())});

keypress()

$("input[type='text']").keypress(function(){ console.log("YOU PRESSED A KEY! YAY"); });

You can listen for a specific keypress as well

$("input").keypress(function(event){ if(event.which===13){ alert("YOU HIT ENTER -- key code 13 is enter key"); } });

on()

Most common event listener used in jQuery

Lets you specifiy specifically which event you want to listen for

$("h1").on("click",function(){ $(this).css("color","purple"); });

$("input").on("keypress",function(){ console.log("Key Pressed!"); });

$("button").on("mouseenter",function(){ $(this).css("font-weight","bold"); });

$("button").on("mouseleave",function(){ $(this).css("font-weight","normal"); });

Why on() instead of click

  • click() only adds listeneres for existing elems
  • on() will add listener for all potential future elems
